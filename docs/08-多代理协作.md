# 08-多代理协作

> 一个网关跑多个独立 AI 助手

## 什么是多代理？

有时候你可能需要：
- 一个 AI 负责工作，一个 AI 负责生活
- 不同平台用不同的 AI
- 某些群聊用特定 AI

OpenClaw 支持同时运行多个独立的 AI 代理。

## 代理类型

### 1. 主代理（Main Agent）

默认的 AI 助手，处理大多数请求。

```yaml
agents:
  main:
    model: anthropic/claude-3-sonnet
    system: 你是一个乐于助人的 AI 助手
```

### 2. 专用代理

针对特定场景的 AI：

```yaml
agents:
  # 编程助手
  coder:
    model: anthropic/claude-3-sonnet
    system: 你是一个专业程序员，擅长各种编程语言
    
  # 写作助手
  writer:
    model: openai/gpt-4
    system: 你是一个专业作家，擅长各种文体
```

## 配置多代理

### 基本配置

```yaml
agents:
  main:
    model: anthropic/claude-3-sonnet
    channels:
      - telegram
      - discord
      
  coder:
    model: anthropic/claude-3-sonnet
    channels:
      - telegram
```

### 按群聊区分

```yaml
agents:
  dev-team:
    model: anthropic/claude-3-sonnet
    chatIds:
      - "-1001234567890"  # 开发群
    system: 你是一个专业的技术顾问
```

## 代理协作

### 1. 转交

一个代理处理不了时，转给另一个。

### 2. 协作

多个代理共同处理一个任务。

## 最佳实践

### 1. 职责清晰

每个代理应该有明确定义的职责，避免重复响应。

### 2. 独立配置

不同代理使用不同的模型和系统提示词。

### 3. 灵活切换

提供便捷的切换方式。
