# 10-安全配置

> Token 保护 / 权限管理 / 安全加固

## 为什么需要安全配置？

你的 AI 助手有：
- 🔑 API Keys（可以操作你的账户）
- 📁 访问文件权限
- 💬 读取你的消息
- 🌐 网络访问能力

一旦泄露，后果严重！

## API Token 保护

### 1. 环境变量（推荐）

不要把 Token 写在配置文件中：

```yaml
# ❌ 错误：直接写死
model:
  apiKey: "sk-xxx"

# ✅ 正确：使用环境变量
model:
  apiKey: "${OPENAI_API_KEY}"
```

运行时指定：

```bash
# Linux/macOS
export OPENAI_API_KEY="sk-xxx"
openclaw start
```

### 2. 使用 .env 文件

```bash
# 创建 .env 文件
cat > ~/.openclaw/.env << EOF
OPENAI_API_KEY=sk-xxx
ANTHROPIC_API_KEY=sk-xxx
EOF

# 限制权限
chmod 600 ~/.openclaw/.env
```

## 权限管理

### 1. 限制对话范围

```yaml
security:
  # 只允许特定用户
  allowedUsers:
    - "telegram:123456789"
    - "feishu:ou_xxx"
  
  # 禁止特定用户
  blockedUsers:
    - "telegram:999999999"
```

### 2. 限制群聊

```yaml
security:
  # 允许的群聊
  allowedChats:
    - "telegram:-1001234567890"
    
  # 私聊模式
  directMessageOnly: true
```

### 3. 敏感操作确认

```yaml
security:
  # 危险操作需要确认
  confirmCommands:
    - "delete"
    - "exec"
    - "settings"
```

## 网络安全

### 1. 本地监听

```yaml
gateway:
  # 只监听本地
  host: "127.0.0.1"
  port: 18789
```

### 2. 防火墙配置

```bash
# Ubuntu/Debian
sudo ufw allow 18789      # 允许 OpenClaw 端口
sudo ufw allow OpenSSH
sudo ufw enable
```

### 3. HTTPS 配置

生产环境建议使用 Nginx 反向代理配置 HTTPS。

## 数据安全

### 1. 敏感信息过滤

```yaml
security:
  # 自动过滤敏感信息
  filterPatterns:
    - "\\d{4}-\\d{4}-\\d{4}-\\d{4}"  # 信用卡
    - "sk-[a-zA-Z0-9]+"              # API Keys
```

### 2. 定期清理

```bash
# 清理旧日志
find logs/ -name "*.log" -mtime +30 -delete
```

## 安全检查清单

部署前确认：

- [ ] API Keys 使用环境变量，不写在配置中
- [ ] 配置文件权限设为 600
- [ ] 防火墙只开放必要端口
- [ ] 禁止公网直接访问（使用 VPN/SSH 隧道）
- [ ] 启用 HTTPS（生产环境）
- [ ] 限制允许的用户/群聊
- [ ] 定期更换 API Keys
